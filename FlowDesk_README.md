## О проекте

Владельцы малого бизнеса и небольшие команды (2–10 человек) теряют продуктивность, распределяя задачи в мессенджерах и таблицах: дедлайны нарушаются, ответственность размыта, прогресс непрозрачен.

**FlowDesk** — лёгкий веб-трекер на основе Kanban-доски, который решает эту проблему без необходимости покупать Jira или Asana. Вся команда начинает работу за 30 секунд: открыл браузер — создал задачу — назначил исполнителя — отслеживаешь прогресс.

```
┌─────────────────────────────────────────────────────────────────┐
│  📋 К выполнению      ⚙️  В процессе        ✅ Выполнено        │
│  ─────────────────    ─────────────────    ──────────────────  │
│  ┌───────────────┐    ┌───────────────┐    ┌───────────────┐   │
│  │ Написать тех- │    │ Код-ревью     │    │ Обновить      │   │
│  │ задание       │    │ PR #12        │    │ README        │   │
│  │ 🔴 Высокий   │    │ 🟡 Средний   │    │ 🟢 Низкий    │   │
│  │ 📅 15.07     │    │ 👤 Иван П.   │    │ 👤 Алексей К.│   │
│  └───────────────┘    └───────────────┘    └───────────────┘   │
└─────────────────────────────────────────────────────────────────┘
```

## Быстрый старт

### Вариант 1 — открыть локально (0 установок)

```bash
# 1. Клонировать репозиторий
git clone https://github.com/your-username/flowdesk.git
cd flowdesk

# 2. Открыть в браузере — вот и всё
open index.html          # macOS
start index.html         # Windows
xdg-open index.html      # Linux
```

> ✅ Поддерживаемые браузеры: **Chrome 90+**, **Firefox 90+**, **Edge 90+**, **Safari 14+**

---

### Вариант 2 — локальный HTTP-сервер

Если браузер блокирует `file://` протокол:

```bash
# Python 3 (встроен в большинство систем)
python3 -m http.server 8080

# Node.js
npx serve .

# PHP
php -S localhost:8080
```

Затем откройте [http://localhost:8080](http://localhost:8080)

---

### Вариант 3 — GitHub Pages

Бесплатный деплой за 3 клика:

1. Сделайте **Fork** этого репозитория
2. Перейдите: `Settings` → `Pages` → `Branch: main` → `/ (root)` → **Save**
3. Через ~60 секунд приложение будет доступно по адресу:
   ```
   https://your-username.github.io/flowdesk/
   ```

## Архитектура

Код организован по принципу **разделения ответственности** — чтобы после MVP легко подключить бэкенд:

```
┌──────────────┐     ┌──────────────────┐     ┌──────────────────┐
│ UIController │────▶│  TaskManager     │────▶│ StorageService   │
│              │     │                  │     │                  │
│ renderBoard()│     │ add(task)        │     │ save(tasks)      │
│ openModal()  │     │ update(id,fields)│     │ load()           │
│ saveTask()   │     │ delete(id)       │     │ clear()          │
│ showToast()  │     │ move(id,status)  │     └──────────────────┘
└──────────────┘     │ filter(criteria) │
                     └────────┬─────────┘
                              │ 1..*
                     ┌────────▼─────────┐
                     │      Task        │
                     │                  │
                     │ id, title, desc  │
                     │ priority, status │
                     │ dueDate, assignee│
                     │ createdAt        │
                     │                  │
                     │ isOverdue()      │
                     │ toJSON()         │
                     └──────────────────┘
```

Когда придёт финансирование — достаточно заменить `StorageService.save/load` на вызовы REST API, и вся остальная архитектура останется нетронутой.
